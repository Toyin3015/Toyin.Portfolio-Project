Using SQL and Power BI to Create an Interactive Dashboard

At a logistics company, upper management struggled with visibility into delivery performance across regions. Reports were coming in late, manually compiled in Excel, and very much inconsistenT. Executives wanted to make faster decisions, but had no real-time data to support them. After interviewing stakeholders, I discovered that the company had a wealth of delivery and driver data stored in a SQL Server database, but no one was using it effectively. I took over the project, and wrote optimized SQL queries to extract key metrics like on-time delivery rate, average delivery duration, and driver performance by route. I then connected Power BI to the SQL database, using scheduled refreshes to ensure up to date data. Within two weeks, using SQL and PowerBi, I had built an interactive dashboard with filters for region, driver, date range, and vehicle type. It included KPIs, heatmaps, and trend lines that could be used for insights.
